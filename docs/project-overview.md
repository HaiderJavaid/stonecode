# Project Overview

## Product Name

Stonecode

## One-Line Description

A paid-beta AI programming tutor where self-taught beginners open persistent course folders and continue coding lessons inside an IDE-like interface.

## Product Concept

Stonecode combines:

- course folders.
- code editor/file structure.
- AI tutor chat.
- course memory.
- subscription-based access.

The user experience should feel like:

> VS Code for learning programming, with an AI tutor that remembers each course.

## Main Screen

The product has two primary states.

### Dashboard State

Right panel shows:

- create new course.
- empty-state prompt for first login.
- guided setup card asking what the user wants to learn.
- favorite learning suggestions.
- finalized courses show Start project before files exist.
- previous course folders after courses exist.
- plan limit status.

Center shows the stone IDE preview/default state.

### Course Workspace State

Left:

- file tree for the selected course.

Center:

- code editor/IDE.

Right:

- course-bound tutor chat.
- course progress.
- resume/detail/progress actions.

## Paid Beta Model

Initial plan:

- Free: 1 active course.
- Basic: 2 active courses.
- Pro: 10 active courses.

Archived courses can stay readable but not actively resumed unless the plan allows it.

## Required SaaS Surfaces

- landing.
- login/signup/password reset.
- dashboard.
- course workspace.
- settings: profile, account, billing, usage.
- privacy, terms, support.

## Design Direction

Preserve:

- dark stone wall background.
- matte stone terminal.
- engraved/inset code typography.
- left glass file panel.
- right course/tutor panel.
- restrained, low-key lighting.

Avoid:

- generic SaaS dashboard look.
- bright card-heavy marketing UI.
- overdecorated gradients.
- replacing the product surface with a landing page.
